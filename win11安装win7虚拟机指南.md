## 准备工作

### 系统要求
- **主机操作系统**: Windows 11
- **内存**: 至少8GB（推荐16GB）
- **硬盘空间**: 至少50GB可用空间
- **CPU**: 支持虚拟化技术（VT-x/AMD-V）

### 检查CPU虚拟化支持
1. 右键点击任务栏，选择"任务管理器"
2. 切换到"性能"选项卡
3. 点击"CPU"
4. 查看"虚拟化"是否显示"已启用"

如果未启用，需要在BIOS中开启：
1. 重启电脑，按F2、F12或Delete进入BIOS
2. 找到"Virtualization Technology"或"Intel VT-x"选项
3. 将其设置为"Enabled"
4. 保存并退出

## 安装VirtualBox

### 安装步骤

1. **运行安装程序**
   - 双击下载的VirtualBox安装包
   - 可能会出现安全警告，点击"是"

2. **安装向导**
   - 点击"Next"进入下一步
   - 选择安装路径（可保持默认）
   - 选择需要安装的组件（建议全部勾选）：
     - [x] VirtualBox Application
     - [x] USB Support
     - [x] Networking
     - [x] Python Support
   - 点击"Next"

3. **网络警告**
   - 会提示安装过程中网络会暂时断开
   - 点击"Yes"继续

4. **开始安装**
   - 点击"Install"开始安装
   - 等待安装完成（约2-5分钟）

5. **完成安装**
   - 安装完成后，取消勾选"Start VirtualBox after installation"
   - 点击"Finish"

6. **重启计算机**（推荐）
   - 安装完成后建议重启电脑
   - 重启后VirtualBox会自动加载必要的驱动程序

### 验证安装

重启后，通过以下方式验证安装是否成功：
- 按Win键，输入"VirtualBox"
- 应该能看到Oracle VM VirtualBox图标

## 创建Windows 7虚拟机

### 1. 启动VirtualBox

1. 双击桌面上的VirtualBox图标
2. 首次启动会显示欢迎界面，点击"OK"关闭

### 2. 创建新虚拟机

1. **点击"新建"按钮**
   - 在VirtualBox主界面，点击"新建"（New）按钮
   - 或者按 Ctrl + N

2. **命名虚拟机**
   - 名称：Windows 7
   - 文件夹：选择虚拟机保存位置（建议放在非系统盘）
   - 类型：Microsoft Windows
   - 版本：Windows 7 (64-bit)
   - 点击"下一步"

3. **分配内存**
   - 根据主机内存分配：
     - 主机8GB内存：分配2-3GB
     - 主机16GB内存：分配4-6GB
     - 主机32GB内存：分配8-12GB
   - 绿色区域表示推荐范围
   - 点击"下一步"

4. **创建虚拟硬盘**
   - 选择"现在创建虚拟硬盘"（Create a virtual hard disk now）
   - 点击"创建"
   - 文件类型：选择"VDI (VirtualBox磁盘映像)"
   - 点击"下一步"

5. **存储方式**
   - 选择"动态分配"（Dynamically allocated）
   - 点击"下一步"

6. **文件位置和大小**
   - 文件位置：选择保存路径（建议非系统盘）
   - 大小：至少40GB（推荐50-60GB）
   - 点击"创建"

### 3. 配置虚拟机设置

1. **选择虚拟机**
   - 在VirtualBox主界面，选择刚创建的"Windows 7"虚拟机
   - 点击"设置"（Settings）按钮

2. **系统设置**
   - 主板 → 启动顺序：确保"光驱"在第一位
   - 处理器 → 处理器数：分配2-4个核心
   - 加速系统：勾选"启用VT-x/AMD-V"和"启用嵌套分页"

3. **存储设置**
   - 存储 → 控制器：IDE
   - 点击"没有盘片"的图标
   - 点击右侧的光盘图标
   - 选择"选择磁盘文件"
   - 浏览并选择下载的Windows 7 ISO文件

4. **网络设置**
   - 网络 → 网卡1：网络地址转换(NAT)
   - 这样虚拟机可以上网

5. **显示设置**
   - 显存：128MB
   - 图形控制器：VBoxSVGA
   - 勾选"启用3D加速"

6. **点击"确定"保存设置**

## 安装Windows 7系统

### 1. 启动虚拟机

1. 在VirtualBox主界面选择"Windows 7"虚拟机
2. 点击"启动"（Start）按钮
3. 虚拟机窗口会打开，显示Windows 7安装界面

### 2. Windows 7安装过程

**步骤1：语言和区域设置**
- 安装语言：中文（简体）
- 时间和货币格式：中文（简体，中国）
- 键盘和输入方法：中文（简体）- 美式键盘
- 点击"下一步" → "现在安装"

**步骤2：启动Windows安装程序**
- 等待安装程序加载文件

**步骤3：接受许可条款**
- 勾选"我接受许可条款"
- 点击"下一步"

**步骤4：安装类型**
- 选择"自定义（高级）"

**步骤5：选择安装位置**
- 选择"驱动器0 分区1"
- 点击"下一步"
- 等待安装程序复制文件和安装系统（约20-30分钟）

**步骤6：首次设置**
- 安装完成后会自动重启
- 输入用户名：例如"Admin"
- 输入计算机名：例如"Win7-VM"
- 点击"下一步"

**步骤7：设置密码（可选）**
- 可以设置密码或留空
- 点击"下一步"

**步骤8：输入产品密钥**
- 如果有产品密钥，输入后激活
- 如果没有，点击"下一步"跳过（试用30天）

**步骤9：Windows更新设置**
- 选择"以后询问我"

**步骤10：时区设置**
- 时区：(UTC+08:00)北京，重庆，香港特别行政区，乌鲁木齐
- 点击"下一步"

**步骤11：网络类型**
- 选择"工作网络"

**步骤12：完成设置**
- 等待Windows完成最终配置
- 进入桌面

### 3. 安装完成后的基本配置

**查看系统信息**
- 右键点击"计算机" → "属性"
- 确认Windows 7已激活或处于试用期

**更改分辨率**
- 右键点击桌面 → "屏幕分辨率"
- 设置合适的分辨率（推荐1024×768或更高）

**关闭用户账户控制（UAC）**
- 控制面板 → 用户账户 → 更改用户账户控制设置
- 拖到最下面（从不通知）
- 点击"确定"

## 安装VirtualBox增强工具

VirtualBox增强工具可以提供以下功能：
- 自动调整虚拟机窗口大小
- 共享剪贴板
- 共享文件夹
- 更好的图形性能
- 时间同步

### 安装步骤

1. **启动Windows 7虚拟机**

2. **安装增强工具**
   - 在VirtualBox菜单栏，点击"设备"（Devices）
   - 选择"安装增强功能光盘映像"
   - 会自动在Windows 7中打开增强工具安装程序

3. **运行安装程序**
   - 双击运行VirtualBox增强工具安装程序
   - 点击"下一步"
   - 点击"安装"
   - 安装过程中会提示安装设备驱动，选择"始终安装此驱动程序软件"
   - 完成后点击"完成"

4. **重启虚拟机**
   - 点击"是"立即重启
   - 重启后增强工具即可生效

### 验证增强工具

重启后，您可以：
- 拖动VirtualBox窗口边缘，Windows 7会自动调整分辨率
- 在主机和虚拟机之间复制粘贴文本和文件

### ⚠️ 错误提示："增强功能包未激活"或"VirtualBox功能增强包似乎在该虚拟机中不可用"

如果看到这个错误提示，说明增强工具没有正确安装或服务没有运行。按以下步骤解决：

#### 解决方案1：检查并启动VirtualBox Guest Additions服务

1. **在Windows 7虚拟机中**：
   - 按 `Win + R`，输入 `services.msc`，按Enter
   - 找到"**VirtualBox Guest Additions Service**"
   - 检查状态：
     - 如果状态为"**已停止**"：
       - 右键点击服务 → 选择"**启动**"
       - 如果启动失败，继续下一步
     - **如果服务不存在**：说明增强工具根本没有安装成功，请直接跳到"解决方案2：重新安装增强工具"

#### ⚠️ 特殊情况：services.msc中找不到VirtualBox Guest Additions Service

如果按照上述步骤在services.msc中找不到"VirtualBox Guest Additions Service"，说明增强工具**根本没有安装成功**。

**快速确认清单**：
- [ ] 控制面板 → 程序和功能中找不到"VirtualBox Guest Additions"
- [ ] 设备管理器中没有任何VirtualBox相关设备
- [ ] services.msc中找不到"VirtualBox Guest Additions Service"
- [ ] 无法自动调整分辨率
- [ ] 无法使用共享文件夹

如果以上都符合，说明增强工具确实未安装，请按以下步骤排查和解决：

**步骤1：确认增强工具是否真的未安装**

检查以下位置，确认增强工具是否已安装：

1. **检查程序和功能**：
   - 控制面板 → 程序和功能
   - 查找"**VirtualBox Guest Additions**"
   - 如果找不到，说明确实未安装

2. **检查设备管理器**：
   - 右键"计算机" → "管理" → "设备管理器"
   - 展开"系统设备"
   - 查看是否有"VirtualBox设备"或相关项
   - 如果完全没有VirtualBox相关项，说明未安装

3. **检查CD驱动器**：
   - 打开"计算机"
   - 查看是否有"CD驱动器 (D:) VirtualBox Guest Additions"
   - 如果还有这个光盘，说明安装程序可能还在，但安装未完成

**步骤2：清理残留文件（如果存在）**

如果发现部分残留，先清理：

1. **卸载残留程序**（如果存在）：
   - 控制面板 → 程序和功能
   - 查找任何包含"VirtualBox"的程序
   - 全部卸载

2. **删除残留文件**（可选）：
   - 在Windows 7中，按 `Win + R`，输入 `%ProgramFiles%\Oracle\VirtualBox Guest Additions`
   - 如果文件夹存在，删除它（需要管理员权限）

3. **重启虚拟机**

**步骤3：重新安装增强工具（从零开始）**

按照以下**完整步骤**重新安装：

**① 确保虚拟机完全启动**
- 等待Windows 7完全加载到桌面
- 等待所有启动项加载完成（约1-2分钟）

**② 挂载增强工具光盘**
- 在VirtualBox菜单：**设备** → **安装增强功能光盘映像**
- 等待5-10秒，让光盘加载

**③ 验证光盘已加载**
- 在Windows 7中，打开"计算机"（或"我的电脑"）
- 应该能看到"**CD驱动器 (D:) VirtualBox Guest Additions**"
- 如果没有看到，尝试：
  - 再次点击"设备" → "安装增强功能光盘映像"
  - 或者手动插入：设备 → 分配光驱 → 选择虚拟盘 → 浏览选择增强工具ISO

**④ 打开CD驱动器**
- 双击CD驱动器图标
- 或者右键点击 → "打开"
- 应该能看到以下文件：
  - `VBoxWindowsAdditions.exe`（主安装程序）
  - `VBoxWindowsAdditions-amd64.exe`（64位版本）
  - `VBoxWindowsAdditions-x86.exe`（32位版本）
  - 其他文件和文件夹

**⑤ 以管理员身份运行安装程序**

⚠️ **关键步骤**：必须使用管理员身份！

1. **找到安装程序**：
   - 在CD驱动器中，找到 `VBoxWindowsAdditions.exe`
   - 或者根据系统选择：
     - Windows 7 64位：使用 `VBoxWindowsAdditions-amd64.exe`
     - Windows 7 32位：使用 `VBoxWindowsAdditions-x86.exe`

2. **右键点击安装程序**：
   - **右键点击** `VBoxWindowsAdditions.exe`
   - 选择"**以管理员身份运行**"
   - ⚠️ **不要直接双击**，必须右键选择管理员身份

3. **处理UAC提示**（如果出现）：
   - 如果提示"是否允许此应用对设备进行更改"
   - 点击"**是**"

**⑥ 处理安装过程中的所有提示**

安装过程中会出现多个提示，**每个都要正确处理**：

1. **欢迎界面**：
   - 点击"下一步"

2. **选择功能**（如果有）：
   - 保持默认选择，点击"下一步"

3. **驱动签名警告**（会出现多次）：
   - 提示："Windows无法验证此驱动程序软件的发布者"
   - **必须选择**："**始终安装此驱动程序软件**"
   - **不要选择**："取消"或"不安装"
   - 这个提示可能会出现3-5次，每次都要选择"始终安装"

4. **安装进度**：
   - 等待安装完成（约2-5分钟）
   - 不要中断安装过程

5. **安装完成**：
   - 点击"完成"
   - 选择"**是，立即重新启动计算机**"
   - ⚠️ **必须重启**，否则增强工具不会生效

**⑦ 重启后验证安装**

重启完成后，按以下步骤验证：

1. **检查服务是否存在**：
   - 按 `Win + R`，输入 `services.msc`
   - 现在应该能看到"**VirtualBox Guest Additions Service**"

## 拖放文件到虚拟机

VirtualBox支持直接拖放文件，这是最快捷的文件传输方式。

### 前提条件

1. ✅ **必须已安装VirtualBox增强工具**（见上一章节）
2. ✅ **虚拟机必须正在运行**

### 方法一：直接拖放文件（推荐）

#### 步骤1：启用拖放功能

1. **启动Windows 7虚拟机**

2. **启用拖放功能**
   - 在VirtualBox菜单栏，点击"设备"（Devices）
   - 选择"拖放"（Drag and Drop）
   - 选择"双向"（Bidirectional）
     - **双向**：主机和虚拟机可以互相拖放文件
     - **主机到虚拟机**：只能从主机拖到虚拟机
     - **虚拟机到主机**：只能从虚拟机拖到主机
     - **已禁用**：关闭拖放功能

   **或者使用快捷键**：
   - 按 **右Ctrl + H**（Host键 + H）
   - 选择"双向"

#### 步骤2：拖放文件

1. **从主机拖到虚拟机**
   - 在Windows 11主机上，找到要传输的文件或文件夹
   - 直接用鼠标**拖拽**文件到Windows 7虚拟机窗口内
   - 松开鼠标，文件会自动复制到虚拟机的桌面或当前文件夹

2. **从虚拟机拖到主机**
   - 在Windows 7虚拟机中，找到要传输的文件
   - 拖拽文件到Windows 11主机的文件夹中
   - 松开鼠标，文件会自动复制到主机

#### 拖放位置说明

- **拖到桌面**：文件会出现在Windows 7桌面
- **拖到文件夹**：文件会复制到该文件夹内
- **拖到资源管理器窗口**：文件会复制到当前打开的文件夹

#### ⚠️ 拖放文件中文乱码问题

如果拖放的文件或文件夹名称包含中文，在虚拟机中显示为乱码，请参考"配置共享文件夹"章节中的"问题：文件夹名称中文乱码"部分，按照解决方案修复编码问题。

**快速解决**：修改Windows 7的区域和语言设置（控制面板 → 区域和语言 → 管理 → 更改系统区域设置 → 选择"中文(简体，中国)" → 重启）。

### 方法二：复制粘贴文件

如果拖放不工作，可以使用复制粘贴：

1. **启用共享剪贴板**
   - 在VirtualBox菜单栏，点击"设备"（Devices）
   - 选择"共享剪贴板"（Shared Clipboard）
   - 选择"双向"（Bidirectional）

2. **复制文件**
   - 在主机上，**右键点击文件** → 选择"复制"
   - 或者按 **Ctrl + C**

3. **粘贴到虚拟机**
   - 在Windows 7虚拟机中，右键点击桌面或文件夹
   - 选择"粘贴"
   - 或者按 **Ctrl + V**

### 拖放功能不工作？解决方法

#### 🔍 快速诊断清单

如果已启用双向但仍无法拖放，按以下清单快速检查：

- [ ] ✅ 已安装VirtualBox增强工具（设备 → 安装增强功能光盘映像）
- [ ] ✅ 已重启虚拟机（安装增强工具后必须重启）
- [ ] ✅ VirtualBox Guest Additions服务正在运行（services.msc中查看）
- [ ] ✅ 使用管理员账户登录Windows 7
- [ ] ✅ 已关闭Windows 7的UAC（用户账户控制）
- [ ] ✅ VirtualBox版本为6.0或更高
- [ ] ✅ 已安装Extension Pack（与VirtualBox版本相同）
- [ ] ✅ 以管理员身份运行VirtualBox

**如果以上都满足但仍无法拖放，请继续查看下面的详细解决方案。**

### 拖放 vs 共享文件夹对比

| 方式 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| **拖放** | 快速、简单、直观 | 需要增强工具，偶尔不稳定 | 传输少量文件 |
| **共享文件夹** | 稳定、支持大文件、双向访问 | 需要配置，占用磁盘空间 | 传输大量文件或频繁访问 |
| **复制粘贴** | 简单、无需配置 | 只能复制，不能移动 | 传输单个文件 |

**推荐**：
- 少量文件：使用**拖放**
- 大量文件或频繁访问：使用**共享文件夹**（见下一章节）

## 配置共享文件夹

共享文件夹允许Windows 7虚拟机访问主机的文件。

### 步骤1：在主机上准备共享文件夹

1. 在Windows 11主机上创建一个文件夹，例如：
   ```
   D:\VMShared
   ```

2. 在文件夹中放入需要共享的文件

### 步骤2：配置VirtualBox共享文件夹

1. **关闭Windows 7虚拟机**（必须先关闭）

2. **打开虚拟机设置**
   - 选择"Windows 7"虚拟机
   - 点击"设置"

3. **添加共享文件夹**
   - 选择"共享文件夹"
   - 点击右侧的"+"按钮
   - 文件夹路径：选择主机的"D:\VMShared"文件夹
   - 文件夹名称：shared
   - 勾选"自动挂载"
   - 勾选"固定分配"（可选）
   - 点击"确定" → "确定"

### 步骤3：在Windows 7中访问共享文件夹

1. **启动Windows 7虚拟机**

2. **打开资源管理器**
   - 双击"计算机"图标

3. **映射网络驱动器**
   - 点击顶部菜单的"映射网络驱动器"
   - 驱动器：选择Z:
   - 文件夹：输入`\\vboxsvr\shared`
   - 勾选"登录时重新连接"
   - 点击"完成"

4. **访问共享文件**
   - 打开"计算机"，会看到Z:驱动器
   - Z:驱动器就是主机的D:\VMShared文件夹

### 另一种访问方式（不映射驱动器）

1. 在Windows 7中打开"运行"（Win + R）
2. 输入`\\vboxsvr\shared`
3. 按Enter即可访问共享文件夹

### ⚠️ 问题：文件夹名称中文乱码

如果在共享文件夹中看到中文文件夹名称显示为乱码（如`????`或`锘挎挱`等），这是编码问题。按以下步骤解决：

#### 解决方案1：修改Windows 7区域和语言设置（推荐）

1. **打开区域和语言设置**：
   - 控制面板 → 区域和语言
   - 或者按 `Win + R`，输入 `intl.cpl`，按Enter

2. **修改格式**：
   - 切换到"格式"选项卡
   - 格式：选择"**中文(简体，中国)**"
   - 点击"应用"

3. **修改位置**：
   - 切换到"位置"选项卡
   - 当前位置：选择"**中国**"
   - 点击"应用"

4. **修改管理**：
   - 切换到"管理"选项卡
   - 点击"更改系统区域设置"按钮
   - 当前系统区域设置：选择"**中文(简体，中国)**"
   - 点击"确定"
   - 会提示重启，选择"**立即重新启动**"

5. **重启后验证**：
   - 重启后，再次访问共享文件夹
   - 中文文件夹名称应该正常显示

#### 解决方案2：修改共享文件夹挂载选项

如果方案1不起作用，尝试修改挂载选项：

1. **关闭虚拟机**

2. **修改共享文件夹设置**：
   - VirtualBox → 选择虚拟机 → 设置 → 共享文件夹
   - 选择已配置的共享文件夹
   - 点击右侧的"编辑"按钮（铅笔图标）
   - 勾选"**只读分配**"（如果需要只读）
   - 点击"确定"

3. **使用命令行挂载（支持UTF-8）**：
   - 启动虚拟机
   - 在Windows 7中，以管理员身份打开命令提示符
   - 使用以下命令挂载共享文件夹：
   ```cmd
   net use Z: \\vboxsvr\shared /persistent:yes
   ```
   - 或者使用VBoxManage命令（在主机上执行）：
   ```cmd
   VBoxManage setextradata "Windows 7" "VBoxInternal2/SharedFoldersEnableSymlinksCreate/Z" 1
   ```

#### 解决方案3：修改注册表（高级方法）

⚠️ **警告**：修改注册表有风险，请先备份注册表。

1. **在Windows 7虚拟机中**：
   - 按 `Win + R`，输入 `regedit`，按Enter
   - 如果提示UAC，点击"是"

2. **导航到以下路径**：
   ```
   HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Nls\CodePage
   ```

3. **修改ACP值**：
   - 找到 `ACP` 项
   - 双击打开
   - 数值数据改为：`936`（简体中文GBK编码）
   - 点击"确定"

4. **修改OEMCP值**：
   - 找到 `OEMCP` 项
   - 双击打开
   - 数值数据改为：`936`
   - 点击"确定"

5. **重启虚拟机**

#### 解决方案4：使用英文文件夹名称（临时方案）

如果以上方法都不行，可以临时使用英文文件夹名称：

1. **在Windows 11主机上**：
   - 将包含中文的文件夹重命名为英文
   - 例如：`文档` → `Documents`，`图片` → `Pictures`

2. **在Windows 7中访问**：
   - 英文文件夹名称不会出现乱码问题

#### 解决方案5：检查VirtualBox版本和设置

1. **检查VirtualBox版本**：
   - VirtualBox菜单：帮助 → 关于VirtualBox
   - 确保使用VirtualBox 6.0或更高版本
   - 旧版本对中文支持可能有问题

2. **检查虚拟机设置**：
   - VirtualBox → 选择虚拟机 → 设置 → 系统 → 主板
   - 确保"扩展特性"中的选项正确

3. **更新VirtualBox**（如果版本过旧）：
   - 访问 https://www.virtualbox.org/
   - 下载最新版本
   - 更新后重新配置共享文件夹

#### 验证修复

修复后，按以下步骤验证：

1. **在Windows 11主机上**：
   - 在共享文件夹中创建一个测试文件夹，使用中文名称，例如：`测试文件夹`

2. **在Windows 7虚拟机中**：
   - 访问共享文件夹（Z:驱动器或`\\vboxsvr\shared`）
   - 查看文件夹名称是否正确显示为"测试文件夹"
   - 如果显示正常，说明问题已解决

3. **测试文件传输**：
   - 在主机上创建一个中文名称的文件，例如：`测试文档.txt`
   - 在虚拟机中查看文件名是否正确显示

#### 常见乱码情况

| 乱码显示 | 可能原因 | 解决方法 |
|---------|---------|---------|
| `????` | 编码不匹配 | 使用解决方案1（修改区域设置） |
| `锘挎挱` | UTF-8 BOM问题 | 使用解决方案2（修改挂载选项） |
| `浣犲ソ` | GBK/UTF-8混用 | 使用解决方案3（修改注册表） |
| 部分乱码 | 字符集不支持 | 使用解决方案4（使用英文名称） |

**推荐解决顺序**：
1. 先尝试**解决方案1**（最简单、最安全）
2. 如果不行，尝试**解决方案2**
3. 如果还不行，尝试**解决方案3**（需要谨慎）
4. 最后考虑**解决方案4**（临时方案）

### ⚠️ 错误提示："增强功能包未激活"或"共享文件夹将无法使用"

如果在配置共享文件夹时看到这个错误，说明VirtualBox增强工具没有正确安装或服务没有运行。

**快速解决方法**：

1. **检查增强工具服务**：
   - 在Windows 7虚拟机中，按 `Win + R`，输入 `services.msc`
   - 找到"VirtualBox Guest Additions Service"
   - 如果状态为"已停止"，右键选择"启动"
   - 如果服务不存在，需要重新安装增强工具

2. **重新安装增强工具**：
   - 参考"安装VirtualBox增强工具"章节中的"错误提示：增强功能包未激活"部分
   - 按照完整步骤重新安装

3. **验证安装**：
   - 重启虚拟机后，再次尝试访问共享文件夹
   - 如果仍提示错误，检查服务是否设置为"自动启动"

**详细解决方案请参考"安装VirtualBox增强工具"章节中的"错误提示：增强功能包未激活"部分。**

## 常见问题解决

### 问题1：虚拟机无法启动，提示"VT-x is disabled"

**解决方案**：
1. 重启电脑，进入BIOS
2. 启用虚拟化技术（Intel VT-x或AMD-V）
3. 保存并退出BIOS
4. 重启后再次尝试启动虚拟机

### 问题2：安装Windows 7时蓝屏

**解决方案**：
1. 关闭虚拟机
2. 进入虚拟机设置
3. 系统 → 加速系统
4. 取消勾选"启用VT-x/AMD-V"和"启用嵌套分页"
5. 重新启动虚拟机安装

### 问题3：虚拟机运行缓慢

**解决方案**：
1. 增加虚拟机内存（不超过主机内存的50%）
2. 增加虚拟处理器数量（2-4个）
3. 在虚拟机内禁用Windows 7的Aero效果：
   - 右键点击桌面 → 个性化
   - 选择"Windows 7基本"主题
4. 确保主机SSD有足够空间
5. 如果主机使用机械硬盘，考虑将虚拟机文件移到SSD上

### 问题4：无法上网

**解决方案**：
1. 检查虚拟机的网络设置
2. 确保网络适配器已启用：
   - 控制面板 → 网络和共享中心 → 更改适配器设置
   - 确保本地连接已启用
3. 尝试桥接模式：
   - 虚拟机设置 → 网络
   - 网卡1：桥接网卡
   - 名称：选择主机的网络适配器

### 问题5：共享文件夹无法访问

**解决方案**：
1. 确保已安装VirtualBox增强工具
2. 检查共享文件夹配置是否正确
3. 在Windows 7中查看网络连接是否正常
4. 尝试重新映射网络驱动器

### 问题6：声音无法播放

**解决方案**：
1. 虚拟机设置 → 音频
2. 勾选"启用音频"
3. 主机音频设备：默认主机音频驱动程序
4. 在Windows 7中检查音量设置

### 问题7：USB设备无法使用

**解决方案**：
1. 下载并安装VirtualBox Extension Pack：
   - 访问https://www.virtualbox.org/wiki/Downloads
   - 下载Extension Pack（与VirtualBox版本相同）
   - VirtualBox → 文件 → 偏好设置 → 扩展 → 添加新包
2. 安装后在虚拟机设置中添加USB过滤器
3. 注意：VirtualBox USB功能需要USB 2.0/3.0控制器

### 问题8：Windows 7激活问题

**注意事项**：
- Windows 7已停止官方支持，不推荐用于生产环境
- 如果需要永久使用，可以：
  1. 购买正版Windows 10/11并升级
  2. 使用Windows 7专业版/企业版的KMS激活（仅限企业用户）
  3. 延长试用期的激活工具（存在风险，不推荐）

## 性能优化建议

### 1. 虚拟机配置优化
- **内存分配**：主机内存的30-50%
- **处理器**：2-4个物理核心
- **虚拟硬盘**：使用动态分配，定期压缩
- **图形内存**：128MB足够日常使用

### 2. Windows 7系统优化
- 禁用不必要的启动项
- 关闭Windows自动更新（因为已停止支持）
- 定期清理临时文件
- 关闭视觉效果
- 禁用系统还原（节省空间）

### 3. 存储优化
- 将虚拟机文件放在SSD上（性能提升显著）
- 定期清理虚拟机垃圾文件
- 使用压缩工具压缩虚拟硬盘（VBoxManage modifyhd命令）

### 4. 网络优化
- 使用NAT模式（默认，安全）
- 需要与其他设备通信时使用桥接模式
- 禁用Windows防火墙（仅在安全环境中）

## 备份和恢复

### 备份虚拟机

**方法1：复制虚拟机文件**
1. 关闭虚拟机
2. 复制整个虚拟机文件夹（包含.vdi文件）
3. 保存到备份位置

**方法2：创建虚拟机快照**
1. 在VirtualBox中选择虚拟机
2. 点击"快照" → "拍摄快照"
3. 输入快照名称和描述
4. 快照占用空间较小，可以随时恢复

**方法3：导出虚拟机**
1. 文件 → 导出设备
2. 选择要导出的虚拟机
3. 选择导出格式（OVF）
4. 完成导出

### 恢复虚拟机

**从快照恢复**：
1. 选择虚拟机
2. 点击"快照"
3. 选择要恢复的快照
4. 点击"恢复快照"

**从备份文件恢复**：
1. 关闭VirtualBox
2. 删除原虚拟机文件夹
3. 将备份的虚拟机文件夹复制到原位置
4. 在VirtualBox中注册虚拟机（文件 → 注册设备）

## 附录：常用命令

### VBoxManage命令行工具

VirtualBox提供了强大的命令行工具VBoxManage，可以在CMD或PowerShell中使用：

**查看虚拟机列表**：
```cmd
VBoxManage list vms
```

**查看运行中的虚拟机**：
```cmd
VBoxManage list runningvms
```

**启动虚拟机（无界面）**：
```cmd
VBoxManage startvm "Windows 7" --type headless
```

**关闭虚拟机**：
```cmd
VBoxManage controlvm "Windows 7" poweroff
```

**查看虚拟机信息**：
```cmd
VBoxManage showvminfo "Windows 7"
```

**压缩虚拟硬盘**：
```cmd
VBoxManage modifymedium disk "D:\VMs\Windows7\Windows7.vdi" --compact
```
